# 自动排版连线优化计划

## 上下文

用户反馈：在自动排版时，当多层级块（如一级块和二级块）同时连接到同一个三级块时，连线会重叠或穿过其他块。当前布局算法没有考虑连线避障。

## 问题分析

### 当前实现

1. **布局算法** (`src/utils/layout.js`):
   - 简单树形布局：根节点居中，子节点向下展开
   - 固定间距：`H_GAP = 60`, `V_GAP = 140`
   - 只考虑块的位置，不考虑连线路径

2. **连线绘制** (`src/canvas.js` 第 880-910 行):
   - 三次贝塞尔曲线：从父节点底部中心到子节点顶部中心
   - 控制点为垂直中点
   - **没有避障逻辑**

### 问题根源

当多个父节点连接到同一个子节点时：
- 每个父节点的连线都从各自底部中心出发
- 所有连线汇聚到子节点顶部中心
- 长距离连线可能穿过中间层级的块

## 解决方案

### 方案一：增加垂直间距（简单快速）

**修改位置**: `src/utils/layout.js:12`

将 `V_GAP` 从 140 增加到 180-200，给连线更多垂直空间。

**优点**: 改动最小，立即见效
**缺点**: 治标不治本，画布会变长

### 方案二：动态垂直间距（推荐）

**修改位置**: `src/utils/layout.js`

根据块的"入度"（有多少父节点连接它）动态调整垂直间距：
- 入度越高的块，与父节点的垂直距离越大
- 公式：`V_GAP = BASE_V_GAP + (inDegree - 1) * EXTRA_GAP`

**优点**: 针对性解决多父节点问题，改动适中
**缺点**: 需要计算入度

### 方案三：连线路由优化（复杂）

修改连线绘制逻辑，实现简单的路由避障：
- 检测连线路径上是否有其他块
- 如有阻挡，调整贝塞尔曲线的控制点

**优点**: 效果最好
**缺点**: 实现复杂，需要大量测试

## 推荐方案：方案二 + 方案一组合

1. **基础垂直间距增加**: `V_GAP = 160` (从 140 提升)
2. **动态调整**: 根据入度增加额外间距
3. **水平间距微调**: `H_GAP = 80` (从 60 提升)

## 关键文件

- `src/utils/layout.js:12` - 调整间距常量
- `src/utils/layout.js:17-30` - `buildTree` 函数，可添加入度计算
- `src/utils/layout.js:81-120` - `layoutSubtree` 函数，应用动态间距

## 验证步骤

1. 创建一个包含多层级共享子节点的测试场景
2. 执行自动排版
3. 检查连线是否仍然重叠或穿过块
4. 调整间距参数直到效果满意
